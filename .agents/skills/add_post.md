# 📝 Agent Skill: Add a New Blog Post

This skill sheet instructs the AI agent on how to create, format, and structure a new blog post for the Hexo ShokaX theme.

---

## 🛠️ Execution Steps

### 1. Generate the Post
Run the standard Hexo generation command in the root folder to create a new markdown file:
```bash
npx hexo new post "your-post-title"
```
This generates a file at:
`source/_posts/your-post-title.md`

### 2. Format the Front-Matter
Open the generated file. ShokaX relies heavily on precise YAML front-matter to render beautiful headers, categories, page music, and tags. Populate the front-matter using this template:

```yaml
---
title: "The Title of Your Post"
date: YYYY-MM-DD HH:mm:ss
tags:
  - Tag1
  - Tag2
categories:
  - ParentCategory
  - ChildCategory
cover: /images/cover-name.jpg
# Optional post-specific background music playlist (e.g. NetEase music)
# If omitted, the global background music from _config.yml will be played.
audio:
  - title: "Song Name"
    list:
      - https://music.163.com/#/song?id=XXXXXX
---
```

#### Field Explanations:
* **`title`**: Keep it engaging. Use quotes to avoid parsing errors.
* **`date`**: Automatically generated by Hexo, but ensure it is in `YYYY-MM-DD HH:mm:ss` format.
* **`tags`**: List of tags. Avoid using too many; 1-3 is ideal.
* **`categories`**: ShokaX styles categories as a breadcrumb hierarchy. If you set:
  ```yaml
  categories:
    - Tech
    - Hexo
  ```
  It will render in the breadcrumb as `Tech > Hexo`.
* **`cover`**: Absolute path to the post cover image. Put the image in `source/images/` and refer to it as `/images/your-image.jpg`.
* **`audio`**: A list of music links for the custom `nyx-player` on the page. You can put a direct NetEase music URL.

---

## 🖋️ Writing Guidelines for Agents

1. **Heading Structure**: Use a proper hierarchical heading structure (start with `##` for sections, `###` for sub-sections). Never use `#` inside the post body as it conflicts with the post title.
2. **Visual Enhancements**:
   * Use standard Markdown tables for data.
   * Highlight key code snippets with syntax highlighting (e.g., ````javascript ... ````).
   * For callouts or alerts, use Hexo tags supported by ShokaX:
     ```html
     <div class="note info">Your informative text here</div>
     <div class="note danger">Your warning/danger text here</div>
     ```
3. **Excerpt Divider**: Use the `<!-- more -->` tag after the first paragraph to separate the post summary/excerpt from the full content. This keeps the home page index neat and tidy.
